# LLM Wiki Knowledge Base — Schema

This file governs how Claude Code maintains this knowledge base. Read this before any wiki operation.

## Ownership Boundaries

| Directory | Owner | Rules |
|---|---|---|
| `raw/` | User | LLM reads only. Never modify, delete, or reorganize. |
| `wiki/` | LLM | LLM creates, updates, merges. User may mark `user-verified`. |
| `outputs/` | LLM | Generated Q&A and reviews. Never fed back into wiki/. |
| `tools/` | Shared | Python utilities. LLM may modify with user approval. |
| `.kb/` | LLM | Internal state tracking. |

## Provenance Model

Every wiki page MUST have a `provenance` field in its frontmatter. Every non-trivial claim within a page MUST cite its source.

### Provenance levels

- **`source-derived`**: Directly extracted or summarized from a file in `raw/`. High trust.
- **`llm-derived`**: Synthesized by LLM across multiple sources. Medium trust. Must cite ≥2 sources.
- **`user-verified`**: Manually confirmed by the user. Highest trust.
- **`query-derived`**: Generated by `/kb-ask`. Low trust. Lives in `outputs/` only.

### Rules

1. `query-derived` content NEVER enters `wiki/` or `index.md`.
2. Synthesis articles (`wiki/synthesis/`) require ≥2 `source-derived` citations.
3. When updating a page, preserve existing `user-verified` content. Only add, don't overwrite verified sections.
4. If a new source contradicts existing wiki content, flag the contradiction explicitly — do not silently overwrite.

## Wiki Page Format

All pages use this frontmatter template:

```yaml
---
title: "Page Title"
type: source | concept | synthesis
provenance: source-derived | llm-derived | user-verified
sources: [citekey1, citekey2]
tags: [keyword1, keyword2]
created: YYYY-MM-DD
updated: YYYY-MM-DD
---
```

### Entity metadata (optional, in tags or dedicated fields)
Instead of separate directories, use tags: `author:hinton`, `method:transformer`, `dataset:imagenet`, `venue:neurips`, `benchmark:glue`. These enable filtering without premature taxonomy.

### Citation format
Inline assertions cite sources at assertion level:
```
The attention mechanism scales quadratically [source: vaswani2017, sec.3.2].
```

### Cross-references
Use Obsidian-compatible wikilinks: `[[Concept Name]]`, `[[Source Slug]]`.

## Operations

### /kb-compile
1. Read `.kb/manifest.json` to find uncompiled files in `raw/`.
2. For each uncompiled file:
   a. Create `wiki/sources/<slug>.md` with summary, key points, metadata. Provenance: `source-derived`.
   b. Identify concepts mentioned. For each concept:
      - If `wiki/concepts/<concept>.md` exists: append new information with source citation.
      - If not: create new concept page. Provenance: `llm-derived`.
   c. Update `wiki/index.md` with new entries.
   d. Mark file as compiled in `.kb/manifest.json`.
3. Append operation to `log.md`.
4. Git commit with message: `compile: <source-slug> (+N sources, +M concepts)`.

### /kb-lint
Run BEFORE any /kb-reflect. Checks:
- [ ] All wiki pages have valid frontmatter (title, type, provenance, sources)
- [ ] All `[[wikilinks]]` resolve to existing pages
- [ ] No `wiki/` page lacks source citations (except user-verified)
- [ ] No duplicate concept pages (fuzzy title match)
- [ ] No orphan pages (not linked from index.md)
- [ ] Source pages reference files that exist in `raw/`

Output: lint report to stdout. Fix automatically where safe, flag ambiguous issues.

### /kb-ask
1. Read `wiki/index.md` for coarse routing.
2. Select ≤10 relevant pages based on query.
3. Read selected pages fully.
4. Synthesize answer with `[source: citekey, location]` citations.
5. Save to `outputs/queries/<timestamp>-<slug>.md` with provenance: `query-derived`.
6. Do NOT update wiki/ or index.md.

### /kb-reflect (Phase 2, requires lint pass)
1. Run `/kb-lint` first. Abort if critical issues found.
2. Read `wiki/index.md` to identify potential cross-cutting themes.
3. Read relevant concept pages.
4. Write synthesis article to `wiki/synthesis/` only if supported by ≥2 source-derived citations.
5. Update index.md.

## Conflict Handling

- If a new source contradicts an existing concept page, add a `## Contradictions` section with both claims cited. Do NOT silently overwrite.
- If two concept pages cover the same topic (fuzzy title match), flag for human review — do NOT auto-merge.
- If a source is retracted or superseded, mark the source page with `status: superseded` in frontmatter and move to `wiki/archive/`. Link the replacement.
- If `/kb-compile` is run on an already-compiled file, skip it (manifest check). Use `--force` to recompile.

## Forbidden Operations

- Never delete a `raw/` file.
- Never overwrite `user-verified` content without explicit user request.
- Never feed `outputs/` content back into `wiki/`.
- Never create wiki pages without provenance frontmatter.
- Never fabricate citations. If unsure, mark with `[UNVERIFIED]`.
- Never merge concept pages without checking for contradictions first.

## Conventions

- Filenames: kebab-case, e.g. `attention-mechanism.md`, `vaswani2017-attention.md`.
- Source slugs: `<firstauthor><year>-<keyword>`, e.g. `vaswani2017-attention`.
- Dates: ISO 8601 (YYYY-MM-DD).
- Language: Match the source language. Default to English for concept pages unless user specifies otherwise.
