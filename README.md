# LLM Wiki

[中文版](README_CN.md)

A personal research knowledge base powered by **Zotero + Claude Code + Obsidian**.

Zotero manages your literature. Claude Code reads, summarizes, cross-references, and maintains a structured wiki. Obsidian gives you a graph-linked viewer. Everything is markdown, version-controlled with git, and governed by a provenance model that tracks where every claim comes from.

## Architecture

```
Raw Sources (you own)  →  Claude Code (LLM engine)  →  Wiki (LLM maintains)
   Zotero papers               /kb-compile                sources/
   Web snapshots               /kb-lint                   concepts/
   Notes & PDFs                /kb-reflect                synthesis/
```

**Three layers:**

1. **Raw Sources** (`raw/`) — Your Zotero library, web snapshots, notes, PDFs. You own this; the LLM only reads.
2. **Wiki** (`wiki/`) — Structured knowledge pages with provenance tracking. The LLM creates and maintains these.
3. **Schema** (`CLAUDE.md`) — Rules governing LLM behavior, provenance levels, and forbidden operations.

**Provenance model** — Every wiki page and every claim carries a trust level:

| Level | Meaning | Trust |
|---|---|---|
| `source-derived` | Directly extracted from a raw source | High |
| `llm-derived` | Synthesized across multiple sources | Medium |
| `user-verified` | Manually confirmed by you | Highest |
| `query-derived` | Generated by Q&A (stays in `outputs/` only) | Low |

## Quick Start

### Prerequisites

- [Claude Code](https://docs.anthropic.com/en/docs/claude-code) CLI installed
- [Zotero](https://www.zotero.org/) with [Better BibTeX](https://retorque.re/zotero-better-bibtex/) plugin (for online sync)
- Python 3.10+ with `pyzotero` (`pip install -r tools/requirements.txt`)
- [Obsidian](https://obsidian.md/) (optional, for viewing the wiki)

### Setup

```bash
git clone https://github.com/<your-username>/llm-wiki.git
cd llm-wiki
pip install -r tools/requirements.txt
```

Open the project in Claude Code:

```bash
claude
```

### Workflow

1. **Sync literature** — `/kb-sync` pulls papers from your Zotero library into `raw/zotero/papers/`
2. **Compile** — `/kb-compile` processes raw files into `wiki/sources/` summaries and `wiki/concepts/` articles
3. **Lint** — `/kb-lint` checks wiki health: broken links, missing provenance, duplicates
4. **Query** — `/kb-ask <question>` searches the wiki and produces cited answers in `outputs/`
5. **Reflect** — `/kb-reflect` generates cross-cutting synthesis articles (requires lint pass first)
6. **View** — Open `wiki/` as an Obsidian vault and explore `[[wikilinks]]`

## Commands

### Core KB Pipeline

| Command | Description |
|---|---|
| `/kb-sync` | Sync Zotero → `raw/zotero/papers/` (online API or offline JSON import) |
| `/kb-ingest` | Stage URLs, notes, documents → `raw/` |
| `/kb-compile` | `raw/` → `wiki/sources/` + `wiki/concepts/` + `index.md` |
| `/kb-lint` | Health check: provenance, wikilinks, duplicates, orphans |
| `/kb-ask` | Query the wiki with cited answers → `outputs/queries/` |
| `/kb-reflect` | Cross-cutting synthesis → `wiki/synthesis/` (post-lint only) |

## Zotero Integration

Two independent sync paths with different failure domains:

| Path | When | How |
|---|---|---|
| **Online** | Zotero running | pyzotero local API (port 23119) |
| **Offline** | Zotero closed | Import Better BibTeX JSON export |

Supports collection-level sync (`--collection "Name"`), WSL2 connectivity auto-detection, and incremental sync via manifest tracking.

## Directory Structure

```
llm-wiki/
├── CLAUDE.md              # Schema: LLM behavior rules + provenance model
├── ARCHITECTURE.md        # Detailed architecture documentation
├── raw/                   # Raw sources (you own, LLM reads only)
│   ├── zotero/papers/     # Auto-synced from Zotero
│   ├── web/               # URL → markdown snapshots
│   ├── notes/             # Freeform notes
│   └── documents/         # PDFs, images, etc.
├── wiki/                  # Knowledge wiki (LLM maintains)
│   ├── index.md           # Master index
│   ├── sources/           # One summary per raw source
│   ├── concepts/          # Topic articles with [[wikilinks]]
│   ├── synthesis/         # Cross-cutting themes
│   └── archive/           # Superseded articles
├── outputs/               # Generated artifacts (NOT fed back into wiki)
│   ├── queries/           # Q&A history
│   └── reviews/           # Literature reviews
├── tools/                 # Python & JS tooling
│   ├── zotero_sync.py     # Zotero sync (two-path)
│   └── requirements.txt   # Python dependencies
├── log.md                 # Append-only activity log
└── .kb/                   # Internal state (manifest, sync tracking)
```

## Design Principles

1. **User owns raw/, LLM maintains wiki/** — clear ownership boundary
2. **Provenance everywhere** — every claim tagged, query-derived never enters wiki
3. **Lint before reflect** — deterministic checks before LLM synthesis
4. **Few directories, rich metadata** — entity types as tags, not premature taxonomy
5. **Incremental & idempotent** — rerunning is safe, only new content processes
6. **Git-backed** — all wiki changes tracked with meaningful commits
7. **Conflicts are flagged, not hidden** — contradictions surfaced explicitly
